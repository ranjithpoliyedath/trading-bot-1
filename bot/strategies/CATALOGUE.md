# Strategy catalogue

All registered strategies live as `BaseModel` subclasses under
`bot/models/builtin/`. Each maps to one `@register_model` class that the
dashboard's model dropdown and the Strategy Finder pick up automatically.

The library was assembled with a quality gate (Phase 3 of the build
plan): each candidate has to clear (a) a peer-reviewed paper or
well-known book chapter as primary source, (b) be implementable on our
6-year (currently ~2-year on disk) daily-bar dataset without
compromise, (c) produce non-zero trades and a positive Sharpe / return
when run on the top-20 most-liquid symbols across the available
history.

Strategies that don't currently clear the gate are still **registered**
so the Strategy Finder can re-tune them — they're just flagged below
as "experimental" with the diagnostic that needs work.

## Experimental / borderline

After re-fetching shallow symbols to a full 5-6yr lookback, two of the
original four "experimental" strategies (`adx_trend_v1`,
`obv_momentum_v1`) cleared the production gate and were promoted.  The
two below remain borderline:

| ID | Source | Validation | Verdict |
|---|---|---|---|
| `donchian_v1` | Faith, *Way of the Turtle* (2007) | +335% return but Sharpe −0.35 | High variance — wins are big but losers come in clusters.  Needs an ATR-based exit instead of fixed time-stop. |
| `keltner_breakout_v1` | Keltner (1960) / Linda Raschke | +280% return but Sharpe −0.25 | Same variance pattern as Donchian.  Try `atr_multiplier` 2.5-3.0 via the Finder. |
| `bollinger_v1` | Bollinger (1980s) | Sharpe 0.50 but −60% return | Sharpe-vs-return mismatch — the strategy snaps back well intraday but takes losses on regime breaks; tighten sentiment gate. |
| `vcp_v1` | Minervini (2013) | 58 trades, −70% return | Pattern is genuinely rare; most "VCP-looking" setups don't follow through. |
| `qullamaggie_v1` | Kullamägi blog/talks | 9 trades, −17% return | Same — too few qualifying setups in the universe.  Either widen criteria or run on a hand-curated growth list. |
| `zscore_reversion_v1` | Avellaneda-Lee (2010) | Sharpe 0.09, −26% return | Got worse with deep data — the long-tail mean-reversion edge that worked in our 2yr sample didn't replicate. |

Each remains registered so the Strategy Finder can keep searching;
`scripts/tune_experimentals.py` is the entry point for batch re-tuning
across the four still-marginal entries.

## Implementation notes

* Every strategy is fully self-contained: `predict_batch` lazily
  appends any indicator columns it needs via `bot.indicators`, so the
  base feature pipeline doesn't have to be re-run when a new strategy
  is added.
* All strategies emit confidence in the 0.50–0.95 band, so the engine's
  `conf_threshold` works consistently across them. Lowering the
  threshold = trading lower-conviction signals.
* Sell signals are *suggestions* — the engine's exit rules
  (`take_profit_pct`, `stop_loss_pct`, `atr_stop_mult`,
  `time_stop_days`, plus `use_signal_exit` for the model's own sell)
  ultimately decide when a position closes.
* Param spaces for the Strategy Finder live in
  `bot/strategy_finder.py::PARAM_SPACES`. Tunables are kept narrow and
  centred on each strategy's published default — we're searching the
  neighbourhood of well-known settings, not from scratch.

## Cross-sectional library

These strategies need to rank across the universe each bar — they
can't be expressed via the per-symbol `BaseModel` interface and run
through their own runner
(`dashboard.backtest_engine.run_cross_sectional_backtest`).

| ID | Name | Source | Validation |
|---|---|---|---|
| `jt_momentum_v1` | Jegadeesh-Titman 12-1 | Jegadeesh & Titman (1993), *Returns to Buying Winners and Selling Losers*, JF | +191.16%, Sharpe 0.64, 57 rebalances over 6yr × 50 symbols |

Cross-sectional models are **excluded from the regular Backtest /
Strategy Finder dropdowns** in the dashboard because they need a
different runner.  Use them via the Python API:

```python
from dashboard.backtest_engine import run_cross_sectional_backtest

out = run_cross_sectional_backtest(
    model_id      = "jt_momentum_v1",
    period_days   = 365 * 6,
    top_decile    = 0.20,        # long top 20%
    rebalance_days= 21,           # ~monthly
    starting_cash = 10_000,
    slippage_bps  = 5,
)
```

## Skipped strategies and why

| Strategy | Reason |
|---|---|
| Antonacci dual momentum | Cross-sectional + needs absolute-momentum filter against bonds; not in our data store yet. |
| PEAD (Post-Earnings Drift) | Needs earnings dates; not in our data store yet. |
| News-shock fade / event-driven | Needs intraday news timestamps + intraday bars. |
| ORB (Opening Range Breakout) | Intraday bars required. |
| Chandelier exit / SuperTrend | These are exit rules, not entry strategies — already exposed as `atr_stop_mult` in the engine. |
